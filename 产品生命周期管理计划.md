# 产品生命周期管理计划

## 1. 概述

本文档详细说明商户管理系统中产品的生命周期管理流程，包括产品状态定义、操作权限、商户状态影响以及技术实现细节。

## 2. 产品状态定义

### 2.1 状态类型

| 状态值 | 中文名称 | 描述 | 前端显示 |
|--------|----------|------|----------|
| active | 在售 | 产品正常销售状态，可在前端展示 | <Tag color="success">在售</Tag> |
| inactive | 下架 | 产品暂时下架，不在前端展示 | <Tag color="warning">下架</Tag> |
| archived | 归档 | 产品已归档，不再进行操作 | <Tag color="default">归档</Tag> |

### 2.2 状态转换

- **创建**：默认状态为 `active`
- **编辑**：可在 `active` 和 `inactive` 之间切换
- **归档**：从任何状态转换为 `archived`

## 3. 产品操作权限

### 3.1 创建产品

**权限要求**：
- 商户状态必须为 `active`
- 用户必须通过身份验证
- 必须提供完整的产品信息（名称、价格、库存等）

**业务限制**：
- 价格必须大于 0
- 库存必须大于等于 0

### 3.2 编辑产品

**权限要求**：
- 用户必须是产品的所有者（验证 merchant_id）
- 用户必须通过身份验证

**可编辑字段**：
- 名称
- 描述
- 价格
- 原价
- 库存
- 状态（active/inactive）

### 3.3 删除产品

**权限要求**：
- 用户必须是产品的所有者（验证 merchant_id）
- 用户必须通过身份验证

**操作流程**：
- 前端显示确认对话框
- 后端验证所有权
- 执行删除操作
- 清除相关缓存

## 4. 商户状态影响

### 4.1 商户状态定义

| 状态值 | 描述 | 对产品管理的影响 |
|--------|------|------------------|
| pending | 审核中 | 无法创建产品，可编辑和删除已有产品 |
| active | 已激活 | 可正常创建、编辑、删除产品 |
| suspended | 已暂停 | 无法创建产品，可编辑和删除已有产品 |
| rejected | 已拒绝 | 无法创建产品，可编辑和删除已有产品 |

### 4.2 商户状态检查

- **创建产品**：后端会检查商户状态，非 `active` 状态的商户无法创建产品
- **编辑/删除**：不受商户状态影响，只要验证所有权即可操作

## 5. 产品生命周期流程

### 5.1 完整流程

1. **创建阶段**：
   - 商户提交产品信息
   - 系统验证商户状态和数据合法性
   - 创建产品，默认状态为 `active`

2. **销售阶段**：
   - 产品在 `active` 状态下正常销售
   - 商户可随时编辑产品信息
   - 商户可将产品改为 `inactive` 状态暂时下架

3. **下架阶段**：
   - 产品处于 `inactive` 状态
   - 不在前端展示，但可重新激活
   - 商户可编辑产品信息

4. **归档阶段**：
   - 产品处于 `archived` 状态
   - 不再进行销售和编辑
   - 保留历史记录

5. **删除阶段**：
   - 产品从系统中完全删除
   - 清除相关缓存
   - 不可恢复

## 6. 技术实现细节

### 6.1 后端实现

- **API 端点**：
  - `POST /api/v1/products` - 创建产品
  - `PUT /api/v1/products/:id` - 更新产品
  - `DELETE /api/v1/products/:id` - 删除产品
  - `GET /api/v1/merchants/products` - 获取商户产品列表

- **权限验证**：
  - 使用 JWT 进行身份验证
  - 验证产品所有权（merchant_id 匹配）
  - 验证商户状态（创建产品时）

- **数据验证**：
  - 价格必须大于 0
  - 库存必须大于等于 0
  - 必需字段验证

### 6.2 前端实现

- **界面功能**：
  - 产品列表展示（支持状态筛选）
  - 产品添加表单
  - 产品编辑表单
  - 产品删除确认

- **状态管理**：
  - 使用 Ant Design 的 Tag 组件展示状态
  - 使用 Select 组件选择产品状态
  - 支持按状态筛选产品

- **错误处理**：
  - 商户状态不满足时的提示
  - 数据验证错误提示
  - 操作成功/失败提示

## 7. 限制条件

### 7.1 业务限制

- **价格限制**：产品价格必须大于 0
- **库存限制**：产品库存必须大于等于 0
- **商户状态限制**：只有 `active` 状态的商户可以创建产品
- **所有权限制**：只能操作自己的产品

### 7.2 技术限制

- **缓存机制**：产品操作后会清除相关缓存
- **分页限制**：产品列表默认每页 20 条，最大 100 条
- **权限验证**：所有操作需要身份验证和权限检查
