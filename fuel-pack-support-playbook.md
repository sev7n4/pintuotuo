# Fuel Pack Support Playbook

## 适用范围

- strict 权益开启
- 加油包走订购路径
- 加油包不可单独购买

## 客服统一口径

1. **为什么不能单独买加油包？**  
   加油包是已开通模型权益的余额补充，不是模型权限本身。为避免“有余额但无可调用模型”的误解，平台要求加油包与在售模型商品或套餐包组合购买。

2. **为什么我有余额仍然调用失败？**  
   strict 权益下，系统会先校验你是否拥有该模型权益，再校验余额是否充足。余额充足但权益不匹配时，仍会被拒绝调用。

3. **哪些订单是合规的？**  
   - 合规：模型 SKU + 加油包  
   - 合规：含模型 SKU 的套餐包（可包含加油包）  
   - 不合规：仅加油包（含拼团、参团、秒杀单买）

4. **如何引导用户补充？**  
   引导前往 `卖场` 或 `模型加油站`，选择“模型商品 + 加油包”的组合方案。

5. **常见错误码怎么解释？**  
   - `FUEL_PACK_PURCHASE_RESTRICTED`：表示加油包未与模型商品组合下单。  
   - `ENTITLEMENT_DENIED`：表示用户对当前请求模型无 strict 权益，即使余额充足也会被拒绝。

## 升级处理建议

- 用户投诉“误导购买”：附上规则截图（结算页提示 + 协议条款）。
- 用户确有权益但仍被拒：升级技术排查 strict 匹配（provider/model）。
- 退款争议：按现行退款条款处理，明确已使用部分不可退。
