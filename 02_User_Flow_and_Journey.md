## B端用户旅程

### B端完整生命周期

```
┌─────────────────────────────────────────────────────────────┐
│ 第一阶段：入驻与认证                                         │
└─────────────────────────────────────────────────────────────┘
    ↓
[了解拼脱脱]
  B端企业：
  • 大模型厂商（智谱、Kimi、MiniMax等）
  • 云服务提供商（阿里云、腾讯云等）
  • 自有模型的初创公司
    ↓
[申请入驻]
  • 访问平台官网
  • 点击"商家入驻"
  • 填写企业信息
    ↓
[资料提交]
  • 营业执照
  • 法人身份证
  • 银行账户
  • API服务说明
  • 联系方式
    ↓
[平台审核] (1-3天)
  • 核实资质
  • 验证API可用性
  • 确认结算账户
    ↓
[审核通过]
  • 店铺激活
  • 获得商户后台权限
  • 开放API Key托管功能

┌─────────────────────────────────────────────────────────────┐
│ 第二阶段：店铺配置与上架                                     │
└─────────────────────────────────────────────────────────────┘
    ↓
[上传API Key]
  B端在商户后台：
  • 上传 API Key 到平台密钥库
  • 设置月度配额上限（如1000万Token）
  • 配置模型版本和参数限制
    ↓
[创建SKU]
  B端定义商品：
  • SKU名称：Kimi K2.5 编码大师包
  • Token数量：100万
  • 零售价：¥100
  • 拼团价规则：2人¥60, 5人¥50
  • 库存上限：1000份/天
    ↓
[商品上架]
  • 保存SKU
  • 设置上架时间
  • 商品在C端首页展示
    ↓
[监控销售]
  • 实时查看销售数据
  • 调整定价或拼团规则
  • 管理库存

┌─────────────────────────────────────────────────────────────┐
│ 第三阶段：销售与数据分析                                     │
└─────────────────────────────────────────────────────────────┘
    ↓
[实时销售数据]
  B端后台显示：
  • 今日销售额
  • 成交笔数
  • 成团率
  • 热销商品排名
    ↓
[用户购买触发的链路]
  C端用户购买：
  • Token被消耗
  • 平台实时计费
  • B端营收增加
  • 消费明细记录
    ↓
[数据看板]
  B端可查看：
  • 商品销售排行
  • 用户复购率
  • 地域分布
  • Token消耗趋势
    ↓
[调整营销策略]
  • 根据数据调整定价
  • 设置促销工具（满减、新客优惠）
  • 参与平台大促活动

┌─────────────────────────────────────────────────────────────┐
│ 第四阶段：结算与拓展                                         │
└─────────────────────────────────────────────────────────────┘
    ↓
[月度结算]
  • 汇总该月的销售数据
  • 扣除平台佣金 (30%-40%)
  • 扣除API调用成本
  • 计算应付给B端的金额
    ↓
[发票与对账]
  • B端查看详细账单
  • 确认应收金额
  • 开具/接收发票
    ↓
[款项转账]
  • 平台通过银行转账
  • 款项到达B端账户
  • 自动记账
    ↓
[持续拓展]
  • 参加平台促销活动
  • 上新更多商品
  • 获得更多曝光
  • 营收稳步增长
```
