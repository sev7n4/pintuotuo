# 权益与计价（产品说明）

## 严格白名单（`ENTITLEMENT_ENFORCEMENT=strict`）

在开启严格模式时，用户通过 **Open API / 代理** 调用某一 `(厂商 provider, 模型 model)` 前，系统会校验：该组合是否被 **已履约的购买订单** 或 **当前有效的订阅** 所覆盖（与 SKU 关联的 SPU 决定覆盖哪些模型）。

- **未覆盖**：请求被拒绝，HTTP **403**，业务错误码 `**ENTITLEMENT_DENIED`**。不会预扣 Token。
- **多条覆盖同一模型**：按 **最近一次履约或续费时间**（锚点）选取 **一条** 权益，并使用其绑定的 **价目快照**（`pricing_version_id`）计算本次调用费用。

「模型」匹配与用量指南接口一致：优先按 SPU 的 `**provider_model_id`**，否则按 `**model_name**`（与扣费逻辑统一）。

## 价目快照与续费

- **一次性购买**：订单履约后，订单上记录 `pricing_version_id`，作为该笔权益的价目锚点。
- **订阅**：订阅行上记录 `pricing_version_id` 与权益锚点时间；**自动续费成功** 后会更新为当前策略下的基准价目版本，并刷新锚点时间，以便「最近续费」规则生效。

用户可在订阅列表接口响应中看到 `pricing_version_id`、`entitlement_anchor_at`（若存在），便于对账与客服解释。

## 与「非严格」模式的区别

默认 `**ENTITLEMENT_ENFORCEMENT=off`**（或未设置）时，不强制上述白名单；扣费仍可按历史逻辑回退到实时价目等路径。上线联调、预发与生产切换前请与研发确认开关策略。

## 用户可见错误（严格模式）


| HTTP | 错误码                     | 含义                               |
| ---- | ----------------------- | -------------------------------- |
| 403  | `ENTITLEMENT_DENIED`    | 当前账户对该 `(provider, model)` 无有效权益 |
| 500  | `PRICING_SNAPSHOT_MISS` | 有权益但价目快照中缺少对应费率（配置异常，应联系运营/研发）   |


具体文案以客户端与 API 文档为准。
