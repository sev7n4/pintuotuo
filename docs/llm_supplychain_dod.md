# LLM 供应链双轨评估与验收口径

## 1. 目标

在不重构现有业务主链路的前提下，完成 LiteLLM 与 OneAPI 的并行评估，形成可量化的选型结论，并定义生产发布门槛。

## 2. 评估对象

- 网关 A：LiteLLM
- 网关 B：OneAPI/New-API
- 上游模型覆盖：OpenAI、Anthropic、国内厂商至少各 1 个代表模型

## 3. 评分维度与权重

| 维度 | 权重 | 说明 |
| --- | --- | --- |
| 兼容性 | 25% | OpenAI 兼容程度、错误语义一致性、SDK 改造成本 |
| 稳定性 | 30% | 成功率、限流恢复能力、故障隔离能力 |
| 成本 | 20% | 单位 token 成本、额外网关资源成本、运维成本 |
| 性能 | 15% | P95/P99 延迟、队列等待、重试放大系数 |
| 运维复杂度 | 10% | 配置复杂度、排障路径、升级风险 |

总分 = 各维度得分 * 权重，满分 100。

## 4. 统一验收指标（DoD）

### 4.1 联通性 DoD

- 双轨网关均能完成 `/v1/chat/completions` 代表模型调用。
- 同一请求在双轨均返回可解析 JSON，错误字段至少包含 `message`、`type`。
- 模型路由映射可由配置管理，不依赖硬编码。

### 4.2 稳定性 DoD

- 注入 429/5xx/timeout 后，系统能在 5 分钟内恢复到基线成功率。
- 熔断触发后可自动半开探测并恢复。
- 请求 trace 可看到策略、候选、重试次数、最终结果。

### 4.3 可观测性 DoD

- Prometheus 可采集 API 成功率、延迟、错误类型、重试次数。
- Grafana 至少具备 3 个看板：API 健康、供应商健康、成本配额。
- Alertmanager 告警链路可用，至少覆盖高错误率和高延迟告警。

### 4.4 质量 DoD

- Promptfoo 基线用例不少于 10 条，覆盖通用问答、格式约束、拒答与异常场景。
- PR 自动跑 prompt 回归，报告可下载。
- 关键场景通过率不低于 80%。

## 5. 发布门槛

- 连续 7 天满足：
  - 成功率 >= 99.0%
  - P95 延迟 <= 3s（按业务可调整）
  - 关键回归用例通过率 >= 80%
- 存在可执行回滚路径（切回原直连路由或备用网关）。

## 6. 建议执行节奏

- 第 1 周：双轨联通 + 稳定性底座修复
- 第 2 周：监控告警 + 评测门禁 + 灰度切换演练
