# 加油包 V1 产品决策摘要（PRD）

> 版本：1.0  
> 状态：已定稿待竞品定价与研发排期  
> 说明：技术载体为现有 SKU 类型 `token_pack`，不新增 SKU 类型枚举。

### 技术实现（批次 FIFO，与代码对齐）

| 项 | 位置 |
|----|------|
| 表结构 | `backend/migrations/056_token_lots.sql`（`token_lots`，历史余额回填为 `legacy_migrated` 无到期批次） |
| 扣减 / 到期 | `backend/billing/token_lots.go`：`DebitLotsFIFO`、`ForfeitExpiredLots`、`CreditTokenLot`；计费引擎扣费前会先处理到期 |
| 履约入账 | `fulfillTokenPack`：加油包等写入带 `expires_at` 的批次；订阅赠送 Token 批次类型 `subscription_bonus`；充值 / 老逻辑为 `CreditLegacyLot`（无到期） |
| API | `GET /api/v1/tokens/lots`（需登录），见 `GetTokenLots` |
| 前台 | 「我的 Token」展示批次表，结算页提示可购 Token 包 |

---

## 1. 产品定义

| 项 | 决策 |
|----|------|
| 名称 | 加油包（Token 补充包） |
| 本质 | 纯 Token/算力补充，**不额外授予模型权限** |
| 与月包关系 | 可单独购买；与月包可同时持有；**扣费顺序：先月包可用额度，后加油包** |
| 技术类型 | 沿用 **`token_pack`**，通过运营配置与规则实现「加油包」产品方案 |

---

## 2. 档位与定价

| 项 | 决策 |
|----|------|
| 档位 | **S / M / L** 三档；支持后台**可配置梯度**（数量、展示名、推荐位等） |
| 定价 | **暂不锁价**，待竞品分析后确定标准价、月包用户价等 |
| 双价机制 | 保留「标准价 / 月包用户价」能力（字段与展示随定价一并落地） |

---

## 3. 有效期与消耗

| 项 | 决策 |
|----|------|
| 默认有效期 | 自购买起 **365 个自然日** |
| 可配置性 | **可配置**（全局或按 SKU/批次策略覆盖） |
| 多批次消耗顺序 | **最早过期优先（FIFO by expiry）** |

---

## 4. 权益与可调用范围

| 项 | 决策 |
|----|------|
| 仅买加油包（无月包） | 可使用 **全部「基础可售」模型**（与当前可售 catalog 一致；若后续有内测/隐藏模型，以产品定义为准） |

---

## 5. 退款

| 项 | 决策 |
|----|------|
| 规则 | **整包未动可退**：该批次入账后 **Token 消耗为 0** 方可按平台规则申请退款 |
| 已消耗 | 已发生消耗则 **不退**（需在客服话术与协议中写清） |

---

## 6. 风控（默认值，均建议可配置）

| 配置项 | 建议默认值 | 说明 |
|--------|------------|------|
| 单日购买次数上限（单用户） | 20 次/天 | 防脚本刷单 |
| 单日购买金额上限（单用户） | ¥5,000 / 天 | 可按客单价调整；大额采购可走对公/人工 |
| 单笔大额二次确认阈值 | ¥1,000 / 笔 | 定价确定后建议对齐为 **L 档价的 1～2 倍** |
| 短时频控（可选） | 10 分钟内 ≤ 5 笔 | 防撞库/盗刷 |

---

## 7. 入口优先级（V1）

建议先上线：

1. **余额不足 / 即将不足拦截**（调用失败或返回余额相关错误时引导购买）  
2. **结算页 / 下单链路加购**（与月包同单多 `order_item` 或明确加购路径）

**首页**作为第二波（活动与品牌曝光，V1.1 与运营位一并考虑）。

---

## 8. 客服与协议

| 项 | 决策 |
|----|------|
| 话术 | 采用已评审的 **购买页规则提示 / 结算确认 / 客服模板**（月包与加油包分列）；上线前法务/运营 **冻结版本号**（如 `CS-TOPUP-v1.0`），后续迭代走小版本 |

---

## 9. 与「套餐包管理」关系

- 加油包**主阵地**在 **`token_pack` SKU** 的配置与上架。  
- **权益包（套餐包）** 为可选：若需在 `/packages` 专区露出「加油包」卡片，可新建权益包且明细为对应 token SKU，**不必**新增 SKU 类型或强制新增「套餐包类型」枚举。

---

## 10. 验收要点（业务）

- 三档可配置、价格待竞品后填入仍可验收「配置与展示」链路  
- 365 天默认、可配置覆盖  
- FIFO 消耗与到期失效行为可验证  
- 仅加油包用户可调用基础可售模型  
- 整包未动可退、已动不退 — 与订单/流水一致  
- 风控默认值生效，可调参
