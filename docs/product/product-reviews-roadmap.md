# 用户评价（UGC）产品与技术路线（供 vNext 定调）

## 背景

商品详情页「用户评价」Tab 当前为占位展示（真实列表依赖后端接口）。是否纳入下一版本需产品决策。

## 建议决策项

| 选项 | 说明 |
|------|------|
| A. vNext 必做 | 完成「订单完成后评价 → 详情页展示」闭环，支撑转化与信任 |
| B. 仅展示运营/聚合分 | 继续仅展示 SPU 聚合评分，不开放用户发文 |
| C. 延后 | 资源不足时明确时间点与依赖（审核、反作弊） |

## 若选 A：建议范围（概要）

1. **后端**: 评价表（order_id、user_id、sku/spu_id、rating、content、status）；列表/创建接口；可选敏感词与审核状态。  
2. **前端**: 订单详情或支付成功页「去评价」入口；详情 Tab 列表与发表表单；空状态与加载/错误态。  
3. **风控**: 已支付/已完成订单限制、同一订单单次评价、可选举报与后台下架。

## 依赖

- 订单状态机与「可评价」条件一致  
- 与现有 `spu_average_rating` / `review_count` 汇总逻辑对齐（定时任务或写入时更新）
