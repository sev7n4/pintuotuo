# P1 多明细订单文案（运营/客服）

## 适用范围

- 订单列表页
- 订单详情页
- 支付页
- 客服 FAQ / 用户通知

## 对外统一表达

- 新版订单已支持「一个订单包含多个商品条目（SKU）」。
- 订单金额为全部条目小计之和。
- 每个条目可在订单详情查看：SKU、数量、小计金额。

## 页面推荐文案

### 订单列表页提示

`订单支持多明细展示：一个订单可包含多个 SKU 条目，详情中可查看每个条目的数量与金额。`

### 订单详情页模块标题

- 标题：`订单明细`
- 说明：`以下为本订单包含的商品条目与对应金额。`

### 支付页说明

- 标题：`订单明细`
- 说明：`支付金额为全部条目合计金额。`

## 客服 FAQ（建议直接复用）

### Q1：为什么现在一个订单里有多条商品？

为了减少重复下单步骤，系统支持把多个商品合并为一个订单统一支付与查看。

### Q2：我如何确认每个商品的金额？

在订单详情中的「订单明细」里可查看每个 SKU 的数量和小计金额。

### Q3：退款是按整单还是按条目？

当前仍按订单流程处理；如需按条目处理，请走人工客服并提供订单号和条目 SKU。
